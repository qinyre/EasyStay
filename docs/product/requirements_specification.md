# 易宿酒店预订平台 - 需求规格说明书 (PRD)

> 本需求文档基于《易宿酒店预订平台》大作业要求整理，旨在为开发团队提供明确的功能边界和业务逻辑准则。

---

## 1. 项目综述

### 1.1 项目目标

本项目旨在为酒店商家与终端消费者之间搭建一个高效、便捷的信息交互桥梁。通过差异化的产品设计，满足不同用户群体的核心诉求。

### 1.2 终端划分

| 终端类型 | 平台 | 核心定位 |
| :--- | :--- | :--- |
| **用户端** | 移动端 | 聚焦用户体验优化，提供简洁直观的预定流程 |
| **管理端** | PC 站点 | 专注于酒店信息管理与运营协作 |

---

## 2. 角色定义

| 角色 | 权限描述 |
| :--- | :--- |
| **终端用户** | 浏览酒店信息、进行预定操作 |
| **商户** | 负责酒店信息的录入、编辑与修改 |
| **管理员** | 负责酒店信息的审核、发布及下线管理 |

---

## 3. 功能需求详述

### 3.1 用户端预定流程（移动端）

#### 酒店查询页（首页）

- **顶部 Banner**：展示酒店广告，点击直接跳转至详情页
- **查询区域**：支持当前地点定位、关键字搜索
- **日历组件**：需开发专门的日历组件用于选择入住日期
- **筛选与标签**：支持星级、价格筛选及快捷标签（如亲子、免费停车场）

#### 酒店列表页

- **条件筛选头**：支持城市、日期、间夜数及搜索设置
- **上滑自动加载**：列表需支持滚动自动加载功能以优化长列表体验
- **信息呈现**：展示酒店名、评分、地址及价格等核心维度

#### 酒店详情页

- **图片 Banner**：支持左右滚动查看酒店图片
- **基础信息**：展示酒店名称、星级、设施及地址
- **房型列表**：房型价格需根据数据实现从低到高的排序

---

### 3.2 管理酒店信息系统（PC 站点）

#### 登录与注册

- 注册时可选择"商户"或"管理员"角色
- 登录界面需根据账号自动判断角色，无需手动选择

#### 酒店信息管理

- **信息录入/编辑**：商户可录入、修改酒店的所有维度数据
- **数据同步**：保存后的数据需确保实时更新到移动端

#### 审核与发布流程

- **状态管理**：包含通过、不通过、审核中三个状态
- **异常处理**：若审核不通过，需显示具体原因
- **下线机制**：支持将酒店下线（虚拟删除），并允许后续恢复

---

## 4. 数据维度规范

### 4.1 必须维度

| 维度类型 | 包含内容 |
| :--- | :--- |
| **基础信息** | 酒店名（中/英双语显示）、酒店地址 |
| **属性信息** | 酒店星级、开业时间 |
| **交易信息** | 酒店房型、酒店价格 |

### 4.2 可选维度

| 维度类型 | 包含内容 |
| :--- | :--- |
| **周边环境** | 附近热门景点、交通及商场信息 |
| **营销信息** | 折扣或优惠场景（如节日 8 折、机票酒店套餐优惠） |

---

## 5. 技术约束与验收标准

### 5.1 技术栈要求

- **前端**：必须使用 **React**
- **后端**：必须使用 **Node.js**

| :--- | :--- | :--- |

| **代码质量** | 10 分 | 项目结构清晰，存储设计合理，具备 README 说明 |
| **用户体验** | 10 分 | 视觉美感、布局合理及多设备兼容性 |
| **创新性** | 10 分 | 新技术应用及自发设计的体验增强点 |
